Privacy

This page describes how the EventFlow demo handles data. It is an example only and not a full privacy policy.

1. What this demo stores

This demo build stores information such as your name, email address, chosen password (as a hashed value), event details, supplier profiles and the content of any messages you send via the Conversations features.

2. Where data is kept

All data for this demo runs locally as part of your development environment. It is not a hosted production service. Removing the demo or clearing the data directory will remove your records.

3. Cookies

EventFlow uses a small authentication cookie so you can stay signed in while you move between pages. The cookie is only used to keep you logged in and is not used for advertising or tracking across other sites.

4. Emails

Some actions in the demo, such as account verification or password reset, attempt to send email. In a local environment these usually log to the console instead of sending real mail. Do not enter sensitive real-world information.

5. Your control

You can delete demo data by removing the local data files or resetting the database using any tooling provided with the project. Because this is a demo, there is no automated export or right-to-erasure workflow.

6. Analytics

This demo does not include third-party analytics scripts. Basic metrics shown in the admin area are generated from the local demo data only.

7. Changes

As EventFlow evolves, these example privacy details may change. Check this page for the latest demo version.
